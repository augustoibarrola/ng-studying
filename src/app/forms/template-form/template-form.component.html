ngForm: Provides information about the current state of the form including a JSON representation of the form value and the validity state of the entire form


In Angular, ngForm is a directive used in a form tag to obtain

  • The values of the various form fields in that respective form
  • Information about the state of a form (i.e., whether it is valid, invalid, etc.)

FormsModule from @angular/forms must be imported in the AppsModule (or whatever module the form's encompassing component is found in) and added to the imports:[] array so that it can be used throughout the components of the application.

ngModel: Provides two-way data binding between the view and component. It is also used to track the state and validity of the input field.

ngSubmit: Fires an event specified by ngSubmit when the form is submitted

#formRef="ngForm" is a template variable that has an instance of ngForm or Angular from. #formRef is thus an ngForm directive instance whcih represents the entire form

(ngSubmit) = "onSubmit()" : ngSubmit is an Angular directive that fires anytime a button of type submit is clicked. In this example, on submitting the form, onSubmit() method gets invoked.

[(ngModel)] = "classVariable": Binds the data entered in the input field of the form to the class variable specified in the corresponding .ts file

#variable = "ngModel": Another template variable which is bound to an instance of ngModel. This helps to track the state of the input field by checking if the user touched the control, if the value changed, or if the value became invalid.
