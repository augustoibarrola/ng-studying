Services

Services allow for the seperation of concerns. It follows the Single Responsibility Principle

A service in Angular is a class that contains some functionality that can be reused across the application. A service is a singleton object.

Angular services come as objects which are wired together using dependency injection.

Services are often used to make HTTP Requests. Servcie classes should handle these requests as well as error inspection, interpretation, and resolution, leaving the component classes to deal with business and view logic.

There is no separate decorator to indicate that a class is a service class

@Injectable() annotation marks a class as a custom-made service class. Service classes should be placed withing the providers:[] attribute of NgModule().

HttpClient

HttpClientModule from @angular/common/http is used to communicate with backend services from an Angular frontend.

HttpClient should be injected into a component class through the its constructor. It'll then offer up methods such as get, post, put, and delete. JSON data that is returned from the backend will be automatically converted to any desired type with these methods.

All HTTP requests are asynchronous and return Observable.

Observable

Observable (from RxJs library) objects can be described as objects that arrive over time. They produce data that must be subscribed to to be used. Observable objects represent asynchronous requests. Sending requests and responses asynchronously increases the performance of Angular applications, and this is done with the help of Observable objects.

Observable.subscribe()

The .subscribe method "listens" for data coming from a given Observable object. It takes three parameters:
  1. a success callback which will be invoked upon successful data retrieval from the Obseravble object.
  2. an error callback which is invoked if an error is returned.
  3. a complete callback which is invoked upon successful streaming of values from Observable.

CORS (Cross Origin Resource Sharing)

CORS refers to any frontend request that is made to a backend that is setup on a different host/port. Unless said backend service is CORS enabled, communcation between the frontend and backend cannot happen. A backend must specify "Origins" are allowed to request resources from it.
