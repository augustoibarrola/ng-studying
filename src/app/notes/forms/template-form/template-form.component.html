Template Forms
Template forms are used for smaller to medium sized forms (i.e., one with only two or three input fields).

ngForm and ngModel directives are used in template forms to get information about the form and it's controls. ngSubmit is used to submit events.

ngForm provides information about the current state of the form including a JSON representation of the form value and the validity state of the entire form

FormsModule from @angular/forms must be imported in the AppsModule (or whatever module the form's encompassing component is found in) and added to the imports:[] array so that it can be used throughout the components of the application.

Form fields marked with a # are known as template variables.

#formRef="ngForm"
#formRef is a template variable.#formRef template variable is then set to ngForm directive.
Thus#formRef is an ngForm directive instance which representing the whole form.

(ngSubmit)="onSubmit()"
ngSubmit is an Angular form-specific directive that fires anytime a submit event happens within the form it populates. In this example, the onSubmit() function inside the component fires everytime the form is submitted.

[(ngModel)]="firstName"
This binds the data entered in that particular <input/> field to a same-named component property in the .ts file.
If [(ngModel)] is not present in the input field, it will not become part of the #formRef ngForm instance.

#firstNameVal="ngModel"
Another template variable (this time in the <input/> field, which binds it to an instance of ngModel. This tracks the state of the input field by checking if the user touched the control, if the value has changed, or if the value has become invalid.
