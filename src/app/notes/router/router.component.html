Router

Routing is the action of navigating between multiple views in a single page. Although Angular SPAs can be built out without routing, routing is useful for:
  • expressing some aspect of the state of the application in the URL

CLI command ng generate guard auth> creates an auth.guard.ts file (and corresonding .spec.ts file), where "auth" is a user defined name for the component.

ActivatedRoute @angular/router ActivatedRoute is an Angular routing service used to access route paramters. An instance of this service will contain all the information on routes associated with the service instance's parent component.
route guards
route guards are the interfaces which communicate to the router on whether a user is allowed to navigate to a requested route or not.
Interface Description
canActivate decides whether a route can be activated or not.
canActivateChild decides whether a child route can be activated or not.
canDeactivate decides whether a route can be deactivated or not.
canLoad decides whether a children can be loaded or not.
Resolve can be implemented be classes to be a data provider, which is used with the router for resolving data while navigating

Preloading

Although lazy loading improves the initial load time of an application, it can cause performance and UX depending on how big a particular module that needs to be loaded is. Preloading is used with Lazy Loading for a better user experience.
Preloading means angular preloads any modules that must be immediately available to the application. When the application compiles, it will compile the main `javascript` bundle required for the initial landing page, and then it will asynchronously load the bundles that have been marked to be loaded in the background.
Preloading happens in two ways:
  1. No modules are preloaded (default)
  2. All modules are preloaded
